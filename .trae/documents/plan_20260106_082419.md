## 修复欢迎界面语言立即切换问题

### 问题分析
- **根本原因**：`SelectedLanguage` 属性的 getter 和 setter 不一致
- **具体表现**：
  - setter 使用 `SetField(ref field, value)` 更新 `field` 变量
  - 但 getter 没有返回 `field` 变量，而是返回默认值
- **导致结果**：当用户选择语言时，setter 被调用，但 UI 无法从 getter 获取正确值
- **最终影响**：语言切换没有立即生效，直到主程序启动后才变更

### 修复方案

1. **修复 `SelectedLanguage` 属性**：
   - 更新 getter 以返回 `field` 变量的值
   - 确保 getter 和 setter 使用同一个 `field` 变量

2. **验证修复效果**：
   - 编译并运行程序
   - 在欢迎界面第一页选择语言
   - 确认界面立即以所选语言显示

### 实施步骤

1. 修改 `WelcomeModel.SelectedLanguage` 属性的 getter
2. 编译项目验证语法正确性
3. 测试语言切换功能
4. 确保欢迎界面立即响应语言变更

### 预期结果
- 用户在欢迎界面第一页选择语言后，界面立即以所选语言显示
- 所有本地化文本（标题、副标题、按钮等）都更新为所选语言
- 语言切换功能正常工作，无需等待主程序启动
